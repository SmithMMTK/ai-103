
AI agent คือบริการซอฟต์แวร์ที่ใช้ generative AI เพื่อทำความเข้าใจและทำงานแทนผู้ใช้หรือโปรแกรมอื่นๆ ต่างจากแอปพลิเคชันแบบดั้งเดิมที่ทำงานตามกฎที่กำหนดไว้ล่วงหน้า AI agent สามารถทำงานได้อย่างอิสระโดยเข้าใจบริบท ตัดสินใจ และลงมือดำเนินการเพื่อให้บรรลุเป้าหมายที่เฉพาะเจาะจง agent เหล่านี้ผสาน AI model ขั้นสูงเข้ากับเครื่องมือเฉพาะทาง เพื่อสร้างระบบอัตโนมัติอัจฉริยะที่ปรับตัวได้ตามสถานการณ์ต่างๆ

วิวัฒนาการของ generative AI ทำให้ agent สามารถทำงานอย่างชาญฉลาดแทนเรา และเปลี่ยนวิธีที่เรานำ AI ไปผสานกับกระบวนการทางธุรกิจและแอปพลิเคชัน การเข้าใจว่า AI agent คืออะไรและใช้งานอย่างไรจึงสำคัญอย่างยิ่งต่อการทำงานอัตโนมัติ การตัดสินใจอย่างมีข้อมูล และการยกระดับประสบการณ์ผู้ใช้

## Why AI agents are useful

AI agents มอบคุณค่าอย่างมากในหลายมิติ:

**Automation of routine tasks** - AI agents จัดการงานซ้ำๆ และงานที่น่าเบื่อ ช่วยให้บุคลากรสามารถโฟกัสกับงานเชิงกลยุทธ์และงานสร้างสรรค์มากขึ้น ส่งผลให้ productivity และ efficiency เพิ่มขึ้นอย่างชัดเจน

**Enhanced decision-making** - ด้วยการประมวลผลข้อมูลจำนวนมหาศาลและสร้าง insight, AI agents ช่วยสนับสนุนการตัดสินใจที่ดีขึ้น โดยสามารถวิเคราะห์แนวโน้ม คาดการณ์ผลลัพธ์ และแนะนำแนวทางจากข้อมูลแบบ real-time ต่างจาก chat model แบบพื้นฐานที่สร้างได้เพียงข้อความ AI agents ใช้ algorithm ขั้นสูงและ machine learning เพื่อวิเคราะห์ข้อมูลและตัดสินใจได้อย่างอัตโนมัติ

**Scalability** - AI agents ช่วยขยายการดำเนินงานได้โดยไม่ต้องเพิ่มทรัพยากรบุคคลในสัดส่วนเดียวกัน องค์กรจึงสามารถเพิ่มขีดความสามารถได้โดยไม่ต้องเพิ่มต้นทุนการดำเนินงานมากนัก

**24/7 availability** - เช่นเดียวกับซอฟต์แวร์ทั่วไป AI agents สามารถทำงานต่อเนื่องได้ตลอดเวลาโดยไม่หยุดพัก ทำให้งานเสร็จทันเวลาและบริการพร้อมใช้งานตลอด 24 ชั่วโมง

## Examples of AI agent use cases

AI agents มีการประยุกต์ใช้ที่หลากหลายในหลายอุตสาหกรรม:

### Personal productivity agents

Personal productivity agents ช่วยงานประจำวัน เช่น การนัดหมายประชุม การส่งอีเมล และการจัดการ to-do list โดย Microsoft 365 Copilot ช่วยผู้ใช้ร่างเอกสาร สร้างงานนำเสนอ และวิเคราะห์ข้อมูลภายในชุด Microsoft Office

### Research agents

Research agents ติดตามแนวโน้ม เก็บรวบรวมข้อมูล และสร้างรายงานอย่างต่อเนื่อง ภาคการเงินใช้เพื่อติดตามผลการดำเนินงานของหุ้น องค์กรด้านสุขภาพใช้เพื่อติดตามงานวิจัยทางการแพทย์ และทีมการตลาดใช้เพื่อวิเคราะห์พฤติกรรมผู้บริโภค

### Sales agents

Sales agents ทำให้กระบวนการ lead generation และ qualification เป็นอัตโนมัติ โดยค้นหา lead ที่มีศักยภาพ ส่งข้อความ follow-up แบบ personalized และนัดหมายการโทรขาย ระบบอัตโนมัตินี้ช่วยให้ทีมขายโฟกัสกับการปิดการขายแทนงานธุรการ

### Customer service agents

Customer service agents จัดการคำถามทั่วไป ให้ข้อมูล และแก้ปัญหาที่พบบ่อย เมื่อนำไปผสานกับ chatbot บนเว็บไซต์หรือแพลตฟอร์มส่งข้อความ จะให้การช่วยเหลือได้ทันที ตัวอย่างเช่น Cineplex ใช้ AI agent เพื่อประมวลผลคำขอคืนเงิน ทำให้ลดเวลาในการจัดการลงอย่างมากและเพิ่มความพึงพอใจของลูกค้า

### Developer agents

Developer agents ช่วยงานด้านพัฒนาซอฟต์แวร์ เช่น code review, bug fixing และ repository management โดยสามารถอัปเดต codebase อัตโนมัติ แนะนำการปรับปรุง และช่วยรักษามาตรฐานการเขียนโค้ด GitHub Copilot คือหนึ่งในตัวอย่างของ agent ประเภทนี้

> [!TIP]
> หากต้องการเรียนรู้เพิ่มเติมเกี่ยวกับ GitHub Copilot ให้ศึกษาจากเส้นทางการเรียนรู้ [GitHub Copilot fundamentals](https://learn.microsoft.com/en-us/training/paths/copilot/)

## Security considerations for AI agents

เมื่อ AI agents มีความเป็นอิสระมากขึ้นและถูกผสานเข้ากับระบบองค์กรมากขึ้น ก็ทำให้เกิดประเด็นด้าน security ที่นอกเหนือจากภัยคุกคามของแอปพลิเคชันแบบดั้งเดิม เนื่องจาก agent สามารถเข้าถึงข้อมูลที่อ่อนไหว ตัดสินใจ และดำเนินการได้เอง คุณจึงต้องออกแบบโดยคำนึงถึง security ตั้งแต่ต้น

ความเสี่ยงด้าน security ที่สำคัญ ได้แก่:

|**Risk Area**|**Description**|**Example Impact**|
|---|---|---|
|**Data leakage and privacy exposure**|Agent มักเข้าถึงข้อมูลธุรกิจหรือข้อมูลผู้ใช้ที่อ่อนไหว หากไม่มีการควบคุมที่เหมาะสม อาจเปิดเผยข้อมูลลับโดยไม่ตั้งใจ|Agent ที่สรุปไฟล์ภายในเผลอใส่ข้อมูลส่วนตัวลงในคำตอบที่ส่งถึงลูกค้า|
|**Prompt injection and manipulation attacks**|ผู้ไม่หวังดีสร้าง input เพื่อ override พฤติกรรมที่ตั้งใจไว้ของ agent หลอกให้เปิดเผยข้อมูลหรือทำ action ที่ไม่ได้รับอนุญาต|คำสั่งที่ซ่อนไว้ในข้อความทำให้ agent รั่วไหล system credentials|
|**Unauthorized access and privilege escalation**|การยืนยันตัวตนหรือ access control ที่อ่อนแอ ทำให้ agent หรือผู้โจมตีที่ควบคุม agent เข้าถึงระบบที่ไม่ควรเข้าถึงได้|Agent ที่เชื่อมกับเครื่องมือ CRM ทำ action ระดับผู้ดูแล เช่น export หรือลบ record|
|**Data poisoning**|ผู้โจมตีทำให้ข้อมูลฝึกสอนหรือข้อมูลบริบทเสียหาย จนทำให้ agent ตัดสินใจแบบมีอคติ ไม่ถูกต้อง หรือไม่ปลอดภัย|dataset ที่ถูกวางยาพิษทำให้ customer support agent แนะนำเนื้อหาที่เป็นอันตราย|
|**Supply chain vulnerabilities**|Agent พึ่งพา external APIs, plugin หรือ model endpoint ซึ่งเพิ่ม attack surface|third-party plugin ที่ถูกเจาะ inject malicious code เข้า workflow ของ agent|
|**Over-reliance on autonomous actions**|agent ที่ autonomous สูงอาจ execute action ที่ไม่ตั้งใจ หากไม่มีการจำกัดหรือ validation อย่างรอบคอบ|agent เผลอส่งการชำระเงินหรือเผยแพร่เนื้อหาที่ยังไม่ตรวจสอบ|
|**Inadequate auditability and logging**|หากไม่มี logging ที่ละเอียด จะติดตาม action หรือพบพฤติกรรมผิดปกติได้ยากตั้งแต่เนิ่นๆ|ทีม security ไม่สามารถระบุการใช้ข้อมูลผิดวัตถุประสงค์ได้เพราะไม่มี activity logs|
|**Model inversion and output leakage**|ผู้โจมตีอาจใช้ output ของ model เพื่ออนุมานข้อมูลอ่อนไหวที่ใช้ระหว่าง training หรือ prompting|การ query ซ้ำๆ ทำให้ดึงข้อมูลส่วนตัวจาก fine-tuning dataset ออกมาได้|

### Mitigation strategies

เพื่อลดความเสี่ยงเหล่านี้ ควรใช้แนวทาง security-by-design ซึ่งประกอบด้วย:

- บังคับใช้ **role-based access controls (RBAC)** และสิทธิ์แบบ **least privilege**
- เพิ่มชั้น **prompt filtering and validation** เพื่อป้องกัน injection attacks
- ทำ sandboxing หรือกำหนดให้การดำเนินการสำคัญต้องผ่าน **human-in-the-loop approvals**
- ดูแลให้มี **comprehensive logging and traceability** สำหรับทุก action ของ agent
- ตรวจสอบ **third-party dependencies** และ integration อย่างสม่ำเสมอ
- retrain และ validate model อย่างต่อเนื่องเพื่อจับสัญญาณ **data drift** หรือ **poisoning attempts**

เมื่อฝังแนวปฏิบัติเหล่านี้ตั้งแต่ช่วงต้นของการพัฒนา คุณจะสามารถ deploy AI agents ได้อย่างปลอดภัยและมั่นใจในสภาพแวดล้อมจริง

## Microsoft Foundry Agent Service overview

Microsoft Foundry Agent Service คือบริการแบบ fully managed ที่ออกแบบมาเพื่อช่วยให้นักพัฒนาสามารถ build, deploy และ scale AI agents คุณภาพสูงได้อย่างปลอดภัย โดยไม่ต้องจัดการทรัพยากร compute และ storage ที่อยู่เบื้องหลัง บริการนี้เปิดให้คุณสร้าง agent ที่ตรงกับความต้องการผ่าน custom instructions และเครื่องมือขั้นสูง

เดิมทีการสร้างประสบการณ์แบบ agent ต้องใช้ความพยายามในการเขียนโค้ดค่อนข้างมากผ่าน API มาตรฐาน Microsoft Foundry Agent Service จัดการความซับซ้อนเหล่านั้นผ่าน interface ที่คล่องตัว ทำให้คุณสร้าง agent ได้ทั้งผ่าน Foundry portal หรือภายในแอปพลิเคชันของคุณเองด้วยโค้ดน้อยกว่า 50 บรรทัด

### Agent types

Microsoft Foundry รองรับ agent หลัก 2 ประเภท:

**Declarative agents** - agent ที่นิยามผ่าน configuration มากกว่าการเขียนโค้ด โดย Declarative agents มี 2 รูปแบบ:

- **Prompt-based agents** - agent เดี่ยวที่กำหนดด้วย model, instructions, tools และ prompts นี่คือประเภทที่พบบ่อยที่สุดและเป็นหัวใจของโมดูลนี้
- **Workflow agents** - การ orchestration แบบ multi-agent ที่นิยามด้วย YAML เพื่อรองรับสถานการณ์ซับซ้อนที่หลาย agent ร่วมมือกันทำงานให้สำเร็จ

**Hosted agents** - agent แบบ containerized ที่สร้างและ deploy ด้วยโค้ด จากนั้นโฮสต์บนแพลตฟอร์ม Foundry โดย Hosted agents ให้คุณควบคุมตรรกะและการประมวลผลของ agent ได้เต็มที่ ขณะที่แพลตฟอร์มเป็นผู้ดูแลโครงสร้างพื้นฐาน

ความเข้าใจ agent แต่ละประเภทช่วยให้คุณเลือกแนวทางที่เหมาะสมกับสถานการณ์ของคุณ โมดูลนี้เน้นไปที่ declarative prompt-based agents เป็นหลัก ซึ่งเป็นเส้นทางเริ่มต้นที่เข้าถึงได้ง่ายที่สุด

### Key features of Microsoft Foundry Agent Service

บริการนี้มีความสามารถสำคัญหลายประการ:

**Automatic tool calling** - บริการดูแล lifecycle ของ tool-calling ทั้งหมด ตั้งแต่การรัน model การเรียกใช้ tools ไปจนถึงการส่งคืนผลลัพธ์ ช่วยลดโค้ด integration ที่ซับซ้อน

**Securely managed data** - สถานะบทสนทนาได้รับการจัดการอย่างปลอดภัยผ่าน Responses API จึงไม่จำเป็นต้องจัดการ state ด้วยตนเอง

**Extensive tool catalog** - ชุด built-in tools และ community tools ที่หลากหลายช่วยขยายความสามารถของ agent ให้มากกว่าการสร้างข้อความ เช่น code execution, file search, web search และการเชื่อมต่อกับ Azure services รวมถึง external APIs

**Model selection** - เลือกใช้ [various AI models](https://learn.microsoft.com/en-us/azure/ai-foundry/agents/concepts/model-region-support?tabs=global-standard#available-models) ให้เหมาะกับความต้องการด้าน performance และ cost

**Enterprise-grade security** - บริการนี้ช่วยรับประกัน data privacy และ compliance ด้วยการจัดการข้อมูลอย่างปลอดภัย keyless authentication และ content safety filters ที่มีมาในตัว

**Customizable storage solutions** - ใช้ storage ที่แพลตฟอร์มจัดการให้ หรือใช้ Azure Blob storage ของคุณเองเพื่อการมองเห็นและการควบคุมที่มากขึ้น

**Observability and tracing** - ความสามารถด้าน monitoring ที่มีมาในตัวช่วยให้คุณติดตามพฤติกรรมของ agent แก้ปัญหา และ optimize performance ใน production

คุณสมบัติเหล่านี้ทำให้การ build และ deploy AI agents เป็นไปอย่างคล่องตัวและปลอดภัยมากขึ้น เมื่อเทียบกับการพัฒนาโดยใช้ Inference API โดยตรง
