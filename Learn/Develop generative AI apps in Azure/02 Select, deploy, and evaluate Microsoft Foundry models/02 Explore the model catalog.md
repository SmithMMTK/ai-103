
แคตตาล็อก Foundry Models เป็นศูนย์กลางสำหรับการค้นหาและเปรียบเทียบโมเดล AI ด้วยจำนวนโมเดลกว่า 1,900 รายการจากผู้ให้บริการหลากหลายราย คุณจึงต้องมีวิธีที่มีประสิทธิภาพในการกรองและค้นหาโมเดลที่ตรงกับความต้องการเฉพาะของคุณ

แคตตาล็อกโมเดลแบ่งได้เป็น 2 กลุ่มใหญ่:

- **Foundry Models ที่ Azure จำหน่ายโดยตรง**
    โมเดลเหล่านี้คิดค่าบริการผ่าน Azure subscription ของคุณโดยตรง และรวมถึงโมเดล Azure OpenAI รวมถึงโมเดลจาก Microsoft และผู้ให้บริการรายอื่น
    
- **Foundry Models จากพาร์ตเนอร์และชุมชน**
    โมเดลเหล่านี้มาจากพาร์ตเนอร์ที่เชื่อถือได้และชุมชน โดยแต่ละโมเดลมีเงื่อนไขไลเซนส์และราคาของตนเอง

## การค้นหาโมเดลในแคตตาล็อกโมเดล

ส่วนติดต่อผู้ใช้ของแคตตาล็อกโมเดลใน Foundry Portal ช่วยให้ค้นหาโมเดลที่เหมาะกับความต้องการของคุณได้ง่าย แต่ละโมเดลมี _model card_ ที่แสดงข้อมูลสำคัญ เช่น ผู้ให้บริการ ความสามารถ เมตริก benchmark ประเด็นด้าน Responsible AI และตัวเลือกการปรับใช้

![Screenshot of the model catalog in Microsoft Foundry portal.](https://learn.microsoft.com/en-us/training/wwl-data-ai/model-catalog-evaluate/media/model-catalog.png)

คุณสามารถค้นหาโมเดลด้วยคีย์เวิร์ด และกรองตามคุณลักษณะต่อไปนี้:

- **Collection**: โมเดลถูกจัดเป็นคอลเลกชัน เช่น โมเดลที่มีให้โดยตรงใน Azure หรือโมเดลในคลัง Hugging Face
- **Capabilities**: ความสามารถเฉพาะของโมเดล เช่น _reasoning_ (การแก้ปัญหาที่ซับซ้อน), _tool calling_ (การเชื่อมต่อ API และฟังก์ชัน), หรือ _multimodal processing_ (ข้อความ รูปภาพ เสียง)
- **Source**: ผู้ให้บริการโมเดล เช่น Azure OpenAI, Microsoft, Cohere, Mistral, Meta, Anthropic และอื่นๆ
- **Inference tasks**: งานเฉพาะ เช่น การสร้างข้อความ การสรุปความ การแปลภาษา การสร้างภาพ การสังเคราะห์เสียงพูด หรือภารกิจ AI ทั่วไปอื่นๆ
- **Fine-tuning methods**: เทคนิคที่รองรับสำหรับการปรับแต่งโมเดล (fine-tuning)
- **Industry**: โมเดลที่ฝึกด้วยชุดข้อมูลเฉพาะอุตสาหกรรม โมเดลเฉพาะทางเหล่านี้มักให้ผลดีกว่าโมเดลแบบใช้งานทั่วไปในโดเมนของตน

## ทำความเข้าใจประเภทโมเดล Generative AI

เมื่อคุณสำรวจแคตตาล็อก คุณจะพบโมเดลหลายหมวดหมู่ที่ออกแบบมาสำหรับกรณีใช้งานต่างกัน โดยภาพรวม คุณสามารถแบ่งโมเดลภาษาได้ดังนี้:

- **Large Language Models (LLMs)** เช่น GPT-5, Mistral Large และ Llama 3 70B ซึ่งออกแบบมาสำหรับงานที่ต้องใช้การให้เหตุผลเชิงลึก การสร้างเนื้อหาที่ซับซ้อน และความเข้าใจบริบทจำนวนมาก โมเดลกลุ่มนี้เหมาะกับแอปที่ซับซ้อน แต่ต้องใช้ทรัพยากรประมวลผลมากกว่า
- **Small Language Models (SLMs)** เช่น Phi-4, โมเดล Mistral OSS และ Llama 3 8B ซึ่งเน้นประสิทธิภาพและความคุ้มค่า พร้อมรองรับงานประมวลผลภาษาธรรมชาติทั่วไปได้ดี เหมาะกับสถานการณ์ที่ความเร็วและต้นทุนสำคัญกว่าการรองรับงานให้เหตุผลที่ซับซ้อนที่สุด และสามารถรันบนฮาร์ดแวร์ระดับล่างหรืออุปกรณ์ edge ได้

### โมเดล Chat completion และโมเดล Reasoning

โมเดลภาษาส่วนใหญ่ในแคตตาล็อกเป็นโมเดล **chat completion** ที่ออกแบบมาเพื่อสร้างข้อความตอบกลับที่สอดคล้องและเหมาะสมกับบริบท โมเดลเหล่านี้เป็นแกนหลักของอินเทอร์เฟซแบบสนทนาและแอปสร้างเนื้อหา

สำหรับงานที่ต้องการประสิทธิภาพสูงในโจทย์ซับซ้อน เช่น คณิตศาสตร์ การเขียนโค้ด วิทยาศาสตร์ กลยุทธ์ และโลจิสติกส์ โมเดล **reasoning** เช่น Claude Opus 4.6 จะมีความสามารถในการแก้ปัญหาที่สูงขึ้น โดยโมเดลสามารถแยกปัญหาซับซ้อนออกเป็นส่วนย่อยและแสดงกระบวนการให้เหตุผลได้

### โมเดลเฉพาะทาง

แคตตาล็อกยังมีโมเดลที่ออกแบบมาสำหรับงานเฉพาะ:

**Embedding models** เช่น Ada และ Cohere ใช้แปลงข้อความให้เป็นตัวแทนเชิงตัวเลข โมเดลกลุ่มนี้รองรับงาน semantic search ระบบแนะนำ และสถานการณ์ Retrieval Augmented Generation (RAG) ที่ต้องค้นหาข้อมูลตามความหมาย ไม่ใช่แค่การตรงคีย์เวิร์ด

**Image generation models** เช่น GPT-image-1 ใช้สร้างภาพจากคำอธิบายข้อความ เหมาะสำหรับสร้างสื่อการตลาด ภาพประกอบ หรือ mockup งานออกแบบ

**Video generation models** เช่น Sora 2 ใช้สร้างวิดีโอจากคำอธิบายข้อความ

**Image analysis models** เช่น GPT-4.1 รองรับอินพุตแบบ _multimodal_ ที่รวมทั้งข้อความและรูปภาพ และสามารถสร้างผลลัพธ์เป็นภาษาธรรมชาติจากพรอมป์ที่มีภาพสำหรับการวิเคราะห์

**Text to speech models** เช่น GPT-4o-tts สามารถแปลงข้อความเป็นเสียงสังเคราะห์

**Speech to text models** เช่น GPT-4o-transcribe สามารถแปลงข้อมูลเสียงพูดให้เป็นข้อความถอดเสียง

### โมเดลเฉพาะภูมิภาคและโดเมน

บางโมเดลถูกปรับแต่งให้เหมาะกับภาษา ภูมิภาค หรืออุตสาหกรรมเฉพาะ เมื่อคุณต้องการประสิทธิภาพเฉพาะทางในโดเมนหรือภาษาหนึ่งๆ โมเดลเหล่านี้มักให้ผลดีกว่าทางเลือกแบบใช้งานทั่วไป ตัวอย่างเช่น โมเดลที่ฝึกจากวรรณกรรมทางการแพทย์ เอกสารทางกฎหมาย หรือคลังข้อมูลภาษาเฉพาะ
